# xiaohongshu-writer

小红书笔记写作 Skill，适用于撰写小红书帖子、种草笔记、教程攻略、观点分享、生活记录等短内容（500-1500字）。

## 什么是 Skill？

Skill 是 Claude Code 的提示词/指令集，可以直接被 Claude Code 调用，让 AI 以特定风格和角色帮助你完成写作任务。

## 快速开始

### 触发方式

当你需要对内容进行小红书风格改写时，直接告诉 Claude，例如：

- "帮我写成小红书风格"
- "发个小红书"
- "用小红书风格写"
- "帮我写篇种草笔记"
- "写个教程攻略"

### 写作流程

```
人：提供素材 + 核心观点 + 个人体验 + 情绪节点
 ↓
AI：标题优化 + emoji推荐 + 结构建议 + 按角度扩写
 ↓
人：二次改写（加入自己的声音、短句化、换行节奏）
 ↓
AI：按四层自检体系检查 → 输出修改建议
 ↓
人：终审和定稿（确认首图策略、互动引导）
```

## 支持的笔记类型

| 类型 | 适用场景 | 字数范围 |
|------|---------|---------|
| 体验报告型 | 调查实验、产品对比 | 800-1200字 |
| 种草/拔草型 | 产品推荐、真实评测 | 600-1000字 |
| 小剧场+洞察型 | 现象解读、场景故事 | 500-800字 |
| 教程攻略型 | 工具分享、步骤教学 | 600-1000字 |
| 经验清单型 | 方法论、避坑指南 | 800-1200字 |

## 核心风格

**"讲人话，像个真实的博主在跟你唠嗑。"**

- 短句为主，大量换行
- emoji 做自然板块分隔符（每篇 3-10 个）
- 口语化表达，大胆说"我觉得"、"我亲测"
- 一句话哲思收尾
- 结尾引导互动（评论/收藏/点赞）

## 质量检查

每次写作完成后，Skill 会进行四层自检：

- **L1 硬性规则**：禁用词、篇幅、emoji 数量、首图策略、互动引导
- **L2 风格一致性**：开头、节奏、口语化、视觉分区
- **L3 内容质量**：干货密度、真实性、一句话哲思、类型专项
- **L4 活人感终审**：温度感、独特性、姿态、评论区测试

## 绝对禁区

1. 超过 1500 字
2. 纯文字无 emoji（每篇至少 3-5 个）
3. 纯文字笔记无首图策略
4. 没有互动引导
5. 使用套话（"首先...其次...最后..."）
6. 假设性例子（必须用真实细节）
7. 空泛表述（"AI工具"→说具体名字）
8. 教科书开头（"在当今XXX发展的时代..."）
